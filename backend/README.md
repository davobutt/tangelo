# Boggle Leaderboard Backend

A Node.js + Express backend service for storing and ranking player scores in the Boggle game. Provides persistent leaderboard rankings, player identity linkage, and score submission endpoints.

## Quick Start

### Development
```bash
npm install
npm run dev
```

Server runs on `http://localhost:3000`

### Production Build
```bash
npm run build
npm start
```

## Features

✅ **Score Submission**: POST endpoint for GUID-keyed score submission with normal or seeded challenge context
✅ **Leaderboard Ranking**: GET endpoint for fetching top scores for normal play or a specific seeded challenge
✅ **Persistent Storage**: SQLite database with indexing for efficient queries
✅ **GUID Merge Rules**: One leaderboard record per GUID with best-score merge behavior
✅ **Mutable Display Name**: Display names can change without breaking score identity
✅ **Data Validation**: Strict payload validation with clear error messages
✅ **Error Handling**: Graceful handling of datastore failures without crashing
✅ **Configuration**: Environment-based setup for local and deployed targets
✅ **CORS Support**: Pre-configured for frontend integration

## Environment Configuration

See `.env.example` for available settings. Key variables:

- `NODE_ENV`: `development` or `production`
- `PORT`: Server port (default 3000)
- `DATASTORE_TYPE`: `sqlite` or `memory` (default sqlite)
- `DB_PATH`: Path to SQLite database (default ./leaderboard.db)
- `LOG_LEVEL`: Logging verbosity (default info)

## API Endpoints

### Health Check
```
GET /health
```
Returns `{ status: "ok", timestamp: ... }`

### Submit Score
```
POST /api/scores
Content-Type: application/json

{
  "playerGUID": "uuid-here",
  "displayName": "Player Name",
  "score": 1450,
  "runMode": "normal"
}
```

Seeded challenge submissions use:

```json
{
  "playerGUID": "uuid-here",
  "displayName": "Player Name",
  "score": 1450,
  "runMode": "seeded",
  "seedKey": "family-night"
}
```

### Fetch Leaderboard
```
GET /api/leaderboard?limit=100
GET /api/leaderboard?limit=100&runMode=seeded&seedKey=family-night
```
Returns top-ranked entries sorted by score descending within the requested run category.

See [API_CONTRACT.md](API_CONTRACT.md) for detailed endpoint documentation.

## Data Model

Each leaderboard entry contains:
- `id`: UUID generated by server
- `playerGUID`: Client-side persistent player identifier
- `displayName`: Human-readable player name
- `score`: Score value (0–1,000,000)
- `submittedAt`: Submission timestamp
- `runMode`: `normal` or `seeded`
- `seedKey`: seeded challenge identifier, or `null` for normal runs

## Datastore

### SQLite (Production)
- File-based persistent storage
- Write-Ahead Logging (WAL) for concurrent access
- Indexed queries for efficient ranking lookups
- Schema auto-initialization on boot

### Memory (Testing)
- In-memory store for integration tests
- Fast iteration, no I/O overhead
- Cleared on shutdown
